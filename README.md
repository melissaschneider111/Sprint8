# Sprint8
# 🧪 Test Case Documentation - Manual QA

Este repositório contém a documentação de testes manuais desenvolvida em Google Sheets, com foco na validação de funcionalidades e garantia da qualidade de software.

---

## 🎯 Objetivo

Aplicar conceitos de Quality Assurance (QA) através da criação e execução de casos de teste estruturados, garantindo que o sistema atende aos requisitos definidos.

---

## 📊 Descrição do Projeto

Os testes foram documentados em formato de planilha, utilizando uma estrutura organizada e padronizada para facilitar a execução, análise e rastreabilidade.

Cada caso de teste inclui informações como:

- Identificador do teste (Test ID)  
- Descrição do cenário  
- Pré-condições  
- Passos de execução  
- Dados de teste  
- Resultado esperado  
- Resultado obtido  
- Status (Pass/Fail)  

Essa abordagem permite maior consistência e clareza durante o processo de validação.

---

## 🔍 Atividades Desenvolvidas

- Criação de cenários e casos de teste com base em requisitos  
- Execução de testes manuais  
- Validação de funcionalidades e fluxos de utilizador  
- Identificação e documentação de bugs  
- Registo de resultados de testes  
- Organização da documentação para fácil manutenção  

---

## 🛠️ Ferramentas Utilizadas
